# Karma Build Tool

`kbuild` is the standardized build and repo orchestration tool for ktools-style
projects. This Rust-local copy keeps the original CMake behavior and adds a
modular Cargo execution path for the Rust workspace.

It is designed to be used as one shared implementation root, typically exposed
on your `PATH` as `kbuild` via a symlink such as:

```bash
ln -s /path/to/kbuild/kbuild.py ~/.local/bin/kbuild
```

## Quick Start

Bootstrapping a new repo from an empty directory:

```bash
kbuild --kbuild-config
# edit ./.kbuild.json
kbuild --kbuild-init
```

Normal day-to-day build commands:

```bash
kbuild --build-latest
kbuild --build-demos
kbuild --clean-latest
```

If the repo uses local `vcpkg`, prepare it first:

```bash
kbuild --vcpkg-install
```

## What Kbuild Manages

- strict validation of the effective kbuild config
- build slots under `build/<slot>/`
- SDK installation under `build/<slot>/sdk`
- ordered demo builds under `demo/*/build/<slot>/`
- repo scaffolding from templates
- optional repo-local `vcpkg`
- git initialization and simple sync flows

## Repository Layout

- `kbuild.py`: underlying entry script, typically exposed on `PATH` as `kbuild`
- `libs/kbuild/`: shared Python implementation
- `templates/`: repo scaffold templates generated by `--kbuild-init`
- `docs/`: hand-written documentation suitable for GitHub Pages
